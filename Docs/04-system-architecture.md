**Use Cases:**
1. **Capture Mushroom Image** - User takes or uploads photo
2. **Answer Observable Traits** - User completes questionnaire about characteristics
3. **Describe Mushroom Features** - User provides text description
4. **Image Recognition** - System classifies using CNN
5. **Trait-Based Classification** - System predicts using ML model
6. **LLM-Based Classification** - System uses language model
7. **Aggregate Predictions** - System combines results
8. **Check for Lookalikes** - System identifies dangerous similar species
9. **View Final Predictions** - User sees ranked species list
10. **Get Species Information** - User views detailed species data

## Key Design Principles

1. **Modularity** - Three independent identification methods can be developed, tested, and improved separately
2. **Scalability** - Components can be deployed independently with load balancing
3. **Robustness** - System continues functioning even if one module fails
4. **Transparency** - Clear explanation of predictions and confidence scores
5. **Safety** - Prominent warnings for dangerous lookalikes
6. **Extensibility** - Easy to add new species, traits, or models
7. **Interoperability** - Well-defined APIs between components
8. **Performance** - Parallel execution of identification modules
